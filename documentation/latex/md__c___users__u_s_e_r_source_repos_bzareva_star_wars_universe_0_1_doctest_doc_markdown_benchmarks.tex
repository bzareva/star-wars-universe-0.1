{\bfseries{doctest}}\+:


\begin{DoxyItemize}
\item is between 0 and 8 times faster than \href{https://github.com/catchorg/Catch2}{\texttt{ {\bfseries{Catch}}}} when using normal expression decomposing {\ttfamily CHECK(a==b)} asserts
\item asserts of the form {\ttfamily \mbox{\hyperlink{doctest__fwd_8h_acab009b596160e614aec1ace5aa8560b}{CHECK\+\_\+\+EQ(a,b)}}} with no expression decomposition -\/ around 31-\/63\% faster than {\ttfamily CHECK(a==b)}

\item using the \href{configuration.md\#doctest_config_disable}{\texttt{ $\ast$$\ast${\ttfamily DOCTEST\+\_\+\+CONFIG\+\_\+\+DISABLE}$\ast$$\ast$}} identifier the asserts just disappear as if they were never written -\/ even lower than the baseline (because most of the implementation is also gone)
\end{DoxyItemize}

\href{https://github.com/catchorg/Catch2}{\texttt{ {\bfseries{Catch}}}}\+:


\begin{DoxyItemize}
\item using \href{https://github.com/catchorg/Catch2/blob/master/docs/configuration.md\#catch_config_fast_compile}{\texttt{ $\ast$$\ast${\ttfamily CATCH\+\_\+\+CONFIG\+\_\+\+FAST\+\_\+\+COMPILE}$\ast$$\ast$}} results in 10-\/30\% faster build times for asserts (and in one case 73\%).
\item using the $\ast$$\ast${\ttfamily CATCH\+\_\+\+CONFIG\+\_\+\+DISABLE}$\ast$$\ast$ identifier provides the same great benefits for assert macros as the doctest version (\href{configuration.md\#doctest_config_disable}{\texttt{ $\ast$$\ast${\ttfamily DOCTEST\+\_\+\+CONFIG\+\_\+\+DISABLE}$\ast$$\ast$}}) -\/ but not for the header cost
